# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с кодом в этом репозитории.

## Язык

Весь код, комментарии, docstring'и, сообщения коммитов и ответы — только на русском языке.

## Обзор проекта

Торговая стратегия на основе новостей для фьючерсов Московской биржи (MOEX). Система собирает новости по нефти и газу из SQLite-баз RSS, генерирует эмбеддинги через локальную Ollama, находит исторически похожие новостные дни по косинусному сходству чанков и предсказывает направление движения цены на следующую сессию. Сейчас настроено для фьючерса на индекс РТС (директория `rts/`), в `.gitignore` заготовки для других инструментов (ng, br, gold, si, spyf, mix).

## Архитектура и пайплайн данных

Пайплайн — последовательность самостоятельных скриптов в `rts/`, каждый читает конфигурацию из `rts/settings.yaml`:

1. **`download_minutes_to_db.py`** — Скачивает минутные свечи с MOEX ISS API в SQLite. Обрабатывает пагинацию, роловер между контрактами, инкрементальное обновление. Выход: БД минутных баров.

2. **`convert_minutes_to_days.py`** — Агрегирует минутные бары в дневные с пользовательскими границами сессии (по умолчанию 21:00 пред. дня — 20:59:59 текущего дня МСК). Обрабатывает роловер контрактов с корректировкой гэпа. Выход: БД дневных баров.

3. **`create_markdown_files.py`** — Читает новости из нескольких `rss_news_*.db`, фильтрует по провайдеру и ключевым словам ("нефт"/"газ"), группирует по торговым сессиям, создаёт по одному `.md` файлу на день. Выход: директория с файлами `YYYY-MM-DD.md`.

4. **`create_embedding.py`** — Генерирует эмбеддинги для каждого дневного markdown через Ollama (модели: `embeddinggemma`, `bge-m3`, `qwen3-embedding`). Разбивает текст на чанки по параграфам с лимитом токенов, L2-нормализует векторы, кэширует в pickle с инкрементальным обновлением по MD5. Выход: `embeddings_ollama.pkl`.

5. **`simulate_trade.py`** — Основной бэктестер. Объединяет дневные котировки с кэшем эмбеддингов, перебирает окна k=3..30, находит наиболее похожий исторический день по косинусному сходству чанков, формирует P/L по совпадению направлений. Выбирает лучшее k по скользящей сумме P/L, применяет инверсию стратегии (зеркало). Выход: `df_rez_output.xlsx`, `explain_topk_all.pkl`, файлы предсказаний.

6. **`strategy_analysis.py`** — Генерирует интерактивный HTML-отчёт через Plotly из `df_rez_output.xlsx`: графики P/L (день/неделя/месяц/год), просадка, скользящие средние, таблицы с метриками стратегии (Sharpe, Sortino, Calmar, PF, RF и др.). Выход: `plots/strategy_analysis.html`.

Вспомогательные скрипты:
- **`check_pkl.py`** — Просмотр содержимого pickle-кэша эмбеддингов.
- **`analyze_explain.py`** — Анализ explain-данных из `explain_topk_all.pkl`, агрегация статистики по заголовкам, формирование white/black-листов.

## Запуск скриптов

Скрипты запускаются по отдельности, строго в порядке пайплайна (1-6):

```bash
python rts/download_minutes_to_db.py
python rts/convert_minutes_to_days.py
python rts/create_markdown_files.py
python rts/create_embedding.py
python rts/simulate_trade.py
python rts/strategy_analysis.py
```

Каждый скрипт читает `rts/settings.yaml` при импорте.

## Конфигурация

Вся конфигурация в `rts/settings.yaml`. Основные параметры:
- `ticker` / `ticker_lc` — код инструмента (например, `RTS`)
- `ticker_close` / `ticker_open` — коды конкретных фьючерсных контрактов (например, `RIM6`)
- `model_name` — модель эмбеддингов Ollama (`embeddinggemma`, `bge-m3`, `qwen3-embedding:0.6b`)
- `provider` — фильтр источника новостей (`investing_prime_interfax`, `investing`, `prime_interfax`)
- `url_ai` — эндпоинт Ollama API (по умолчанию `http://localhost:11434/api/embeddings`)
- Внешние пути (`md_path`, `predict_path`, `path_db_minute`, `path_db_day`, `db_news_dir`) используют плейсхолдеры `{ticker}` / `{ticker_lc}`, подставляемые в рантайме

## Зависимости

- **Ollama** должна быть запущена локально с загруженной моделью эмбеддингов
- **MOEX ISS API** — публичный, без авторизации, используется для рыночных данных
- Внешние SQLite-базы с новостями (`rss_news_*.db`) ожидаются в `db_news_dir`
- Python-зависимости: `pip install -r requirements.txt`
